ეს პროექტი აგებულია სახის გამომეტყველების კლასიფიკაციას (Facial Expression Recognition) **FER2013** მონაცემთა ბაზაზე.

პროექტის მთავარი მიზანია არა უბრალოდ საბოლოო მაღალი სიზუსტის მიღწევა, არამედ კვლევითი და იტერაციული მიდგომის ჩვენება: მარტივი არქიტექტურიდან (Baseline) დაწყებული, ზედმეტად მორგების (Overfitting) პროვოცირება და შემდგომ მისი გასწორება რეგულარიზაციის მეთოდებით.

## ექსპერიმენტების ეტაპები და არქიტექტურული ანალიზი

### ეტაპი 1: საბაზისო მოდელი (`baseline`)

შევქმენით სტანდარტული, შედარებით მარტივი Convolutional Neural Network (CNN) არქიტექტურა რეგულარიზაციის ფენების (Dropout, BatchNorm) გარეშე. მოდელი ვიზუალური ნიშნების ექსტრაქციისთვის იყენებს რამდენიმე საკონვოლუციო ფენას, MaxPool-სა და ბოლო Fully Connected (FC) შრეს კლასიფიკაციისთვის.

| პარამეტრი | მნიშვნელობა |
| --- | --- |
| **Epochs** | 10 |
| **Learning Rate** | 0.001 |
| **Batch Size** | 128 |

#### რატომ და რა შედეგი მივიღეთ:

მიზანი იყო მოდელის საწყისი პოტენციალის (Capacity) შეფასება. მოდელმა აჩვენა სტაბილური სწავლება — Train Loss შემცირდა 1.71-დან 1.30-მდე, ხოლო Validation Accuracy-მ **51.44%**-ს მიაღწია მე-10 ეპოქაზე. `grad_norm` Wandb-ზე ვლოგავდით (per-layer) ყოველ ეპოქაზე forward/backward pass-ის სისწორის შესამოწმებლად.

#### კლასიფიკაციის ანალიზი (Baseline):

* **Happy (F1: 0.71):** ყველაზე მაღალი სიზუსტე — კლასი მონაცემებში ყველაზე მეტია (Support: 1471) და ღიმილი სტრუქტურულად ადვილად ამოსაცნობია.
* **Disgust (F1: 0.12):** ძალიან დაბალია, რადგან კლასი უკიდურესად დისბალანსურია (მხოლოდ 75 სურათი ვალიდაციაზე). მოდელს უჭირს მცირე კლასის გენერალიზაცია.


### ეტაპი 2: ზედმეტი გადასწავლების მოდელი (`overfit`)

#### რა გავაკეთეთ:

ხელოვნურად გავზარდეთ მოდელის ტევადობა (Capacity) — დავამატეთ უფრო ღრმა საკონვოლუციო შრეები და ფილტრების რაოდენობა, ყოველგვარი შეზღუდვის გარეშე.

| პარამეტრი | მნიშვნელობა |
| --- | --- |
| **Epochs** | 20 |
| **Learning Rate** | 0.001 |
| **Batch Size** | 128 |

#### რატომ და რა შედეგი მივიღეთ:

მიზანი იყო იმ თეორიული წერტილის პოვნა, სადაც მოდელი წყვეტს რეალური კანონზომიერებების სწავლას და გადადის "ზეპირებაზე". შედეგი არის სახელმძღვანელოსებრი Overfitting-ის მაგალითი. `grad_norm` Wandb-ზე ვლოგავდით (per-layer) ყოველ ეპოქაზე:

* Train Loss სტაბილურად მცირდებოდა და დაეცა **0.58**-მდე.
* Validation Loss-მა მინიმუმს მიაღწია მე-9 ეპოქაზე (1.1370), რის შემდეგაც დაიწყო მკვეთრი ზრდა და მე-20 ეპოქისთვის ავიდა **1.4241**-ზე.

#### ინტელექტუალური დასკვნა:

ეს დივერგენცია (როცა Train Loss აგრძელებს კლებას, მაგრამ Val Loss იწყებს ზრდას) არის კლასიკური ნიშანი იმისა, რომ მოდელმა დაიწყო სატრენინგო სურათების პიქსელური ხმაურის მემორიზაცია. მიუხედავად იმისა, რომ `val_acc` 57.95%-მდე ავიდა, მზარდი Validation Loss მიუთითებს მოდელის არასტაბილურობაზე ახალ მონაცემებთან მიმართებაში.

---

### ეტაპი 3: რეგულარიზებული მოდელი (`regularized`)

#### რა გავაკეთეთ:

ზედმეტი გადასწავლების პრობლემის მოსაგვარებლად, არქიტექტურაში ჩავაშენეთ სტრუქტურული შეზღუდვები:

* **Batch Normalization** — სტაბილური გრადიენტების დინებისთვის და შრეების ნორმალიზაციისთვის.
* **Dropout** — ნეირონების ნაწილის შემთხვევითი გათიშვა ყოველ ბეტჩზე, რათა ქსელმა არ განავითაროს კო-ადაპტაცია და არ დაეყრდნოს კონკრეტულ პიქსელებს.

| პარამეტრი | მნიშვნელობა |
| --- | --- |
| **Epochs** | 10 |
| **Learning Rate** | 0.001 |
| **Batch Size** | 128 |

#### რატომ და რა შედეგი მივიღეთ:

მიზანი იყო loss-ლანდშაფტის გასუფთავება და სტაბილურობის მიღწევა. შედეგად, overfit მოდელის ვალიდაციის დანაკარგის ზრდა ალაგმდა. Val Loss სტაბილურად დაეცა და გაჩერდა **1.23**-ის ფარგლებში (ყველაზე დაბალი და სტაბილური მაჩვენებელი სამივე მოდელს შორის).

####  მნიშვნელოვანი შენიშვნა სიზუსტეზე (54.00%):

მიუხედავად იმისა, რომ `val_acc` ოდნავ ჩამოუვარდება Overfit მოდელს, რეგულარიზებული მოდელები ბევრად უფრო ნელა სწავლობენ, რადგან Dropout მუდმივად "უშლის ხელს" მარტივ მემორიზაციაში. მისი 1.23 Val Loss ადასტურებს, რომ ეს მოდელი სტრუქტურულად ჯანსაღია და მეტი ეპოქის პირობებში რეალური გენერალიზაციით აჯობებს სხვებს.

####  კლასიფიკაციის ანალიზი და პარადოქსი (Regularized):

* **Sad (Recall: 0.71):** რეგულარიზებულმა მოდელმა საკმაოდ კარგად იპოვა სევდიანი სახეების 71%. თუმცა Precision დაბალია (0.34) — როცა მოდელი ყოყმანობს, იგი ხშირად ირჩევს "Sad" კლასს, რაც კლასების სტრუქტურული მსგავსებებითაა განპირობებული.

## დასკვნა

ექსპერიმენტების სერიამ ნათლად აჩვენა deep learning-ის ფუნდამენტური პრინციპი: მოდელის პარამეტრების ბრმად გაზრდა იწვევს მონაცემთა ზეპირებას (`overfit`), ხოლო სტრუქტურული შეზღუდვების (`BatchNorm`/`Dropout`) შემოტანა აუცილებელია რეალური გენერალიზაციის მისაღწევად — თუნდაც ეს საწყის ეპოქებზე უფრო ნელ კონვერგენციას იწვევდეს.
